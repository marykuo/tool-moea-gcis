# tool-moea-gcis

[![python](https://img.shields.io/badge/python-3.9+-blue.svg)](https://www.python.org/)
[![uv](https://img.shields.io/endpoint?url=https://raw.githubusercontent.com/astral-sh/uv/main/assets/badge/v0.json)](https://github.com/astral-sh/uv)

Python package for fetching and updating GCIS (MOEA) COD data via API.

## Features

- Automated Fetching
- Structured Output: 整合並轉換為結構化 JSON 檔案，便於後續使用。

## Supported Data

- [公司行號及有限合夥營業項目代碼表檢索系統](docs/moea-gcis-spec.md)：營業項目分類代碼（COD）

## Quick Start

This project is managed using [uv](https://github.com/astral-sh/uv). Follow the commands below to get started:

```bash
# Install dependencies and update the lockfile
uv sync

# Execute data update
uv run main

# Linting and automatic formatting
uv run ruff check
uv run ruff format
```

## Project Structure

```
tool-moea-gcis/
├── pyproject.toml                 # Project metadata and dependencies
├── uv.lock                        # uv lockfile (auto-generated)
├── README.md                      # Project documentation
├── data/                          # Output data (JSON files)
│   └── cod/
├── docs/                          # Documentation files
└── src/                           # Source package
    └── moea_gcis/                 # Core logic package
        ├── __init__.py
        ├── main.py                # Entry point: fetch and update GCIS cod data (Wrapper)
        ├── core/                  # Utility modules
        └── scrapers/              # Scraper implementations
```

## Contributing Guidelines

- Use `uv` for all dependency management and environment creation
- Do not edit files in `data/` manually; they are generated by the scripts
- Keep credentials and secrets out of the codebase
- For reproducibility, always commit `pyproject.toml` and `uv.lock`
